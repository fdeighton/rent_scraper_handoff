# Design Principles

These standards generally prioritize:
- clarity over cleverness
- explicitness over hidden behavior
- maintainability over short-term optimization
- predictable patterns over unnecessary abstraction
- operational visibility over opaque workflows

# Final Principle

Code should remain understandable, maintainable, observable, and safe to evolve over time.
